# AI 分析与决策模块 PRD

## 1. 模块目标

让 TeamFlow 在稳定执行基础动作之外，具备基于项目数据做结构化分析、风险判断、建议生成和受控自动决策的能力。

## 2. 模块定位

本模块不是独立产品入口，而是整个 TeamFlow 的智能中枢。它为提醒、汇报、建议和后续动作提供判断能力，但不能绕过平台审计和权限控制。

## 3. 模块边界

### 3.1 本模块负责

1. 规则层判断。
2. 策略层匹配。
3. AI 分析调用。
4. 模型层级路由。
5. 自治级别判断。
6. 输出格式约束。
7. AI 失败降级。
8. 决策记录。

### 3.2 本模块不负责

1. 飞书消息接入。
2. 外部动作直接执行。
3. 数据采集。
4. 高风险动作绕过审批。
5. 自动删除、移除成员或关闭项目。

## 4. 能力层次

### 4.1 规则层

处理确定性强、可直接判断的事件：

1. 任务逾期。
2. 任务阻塞。
3. 分支停滞。
4. 定时生成站会摘要。
5. 定时生成周报。

规则层输出应稳定、低成本、可解释。

### 4.2 策略层

处理较复杂但仍可模板化表达的场景：

1. 成员过载建议重分配。
2. 不同风险等级下的通知策略。
3. 多个风险聚合成摘要。
4. 重复提醒降噪。

策略层必须有明确输入条件和输出动作建议。

### 4.3 AI 分析层

处理规则和策略无法覆盖的分析型场景：

1. 项目整体风险归因。
2. 多因素风险分析。
3. 周报自然语言总结。
4. 自然语言问答。
5. 复杂行动建议。

AI 分析层不得直接执行高风险动作。

## 5. 输入数据

1. 项目基础信息。
2. 任务状态。
3. 成员负载。
4. Git 活动。
5. 历史观察记录。
6. 历史决策记录。
7. 用户提问上下文。
8. 可用动作列表。
9. 自治级别规则。

## 6. 输出结果

1. 结构化提醒内容。
2. 风险分析报告。
3. 站会摘要。
4. 周报。
5. 任务建议。
6. 需要审批的动作建议。
7. 自然语言回复。
8. 决策记录。

## 7. 决策机制

默认顺序：

1. 优先匹配规则。
2. 其次匹配策略。
3. 最后调用 AI 分析。

命中规则：

1. 规则层能给出明确结论时，不调用 AI。
2. 策略层能覆盖场景时，只在需要自然语言组织时调用低成本模型。
3. AI 层必须拿到足够上下文，缺少关键数据时输出数据缺口。

## 8. 自治级别

### 8.1 auto

适用于低风险动作：

1. 发送通知。
2. 生成摘要。
3. 发送提醒。
4. 写入观察记录。

### 8.2 approval

适用于需要人工确认的动作：

1. 建议重分配任务。
2. 更改任务优先级。
3. 更新重要文档。
4. 通知特定高层或外部成员。

### 8.3 forbidden

适用于禁止自动执行的动作：

1. 删除关键数据。
2. 移除成员。
3. 关闭项目。
4. 修改审计记录。
5. 输出或转发密钥。

## 9. 模型层级

### 9.1 fast

适用：

1. 简单摘要。
2. 命令响应。
3. 简短文本改写。

### 9.2 smart

适用：

1. 风险分析。
2. 周报。
3. 站会摘要。
4. 多项数据总结。

### 9.3 reasoning

适用：

1. 多约束判断。
2. 策略生成。
3. 复杂根因分析。
4. 需要权衡多个行动方案的场景。

## 10. 输出规范

所有 AI 输出必须包含或能映射到以下结构：

1. 事实依据。
2. 判断结论。
3. 风险等级。
4. 建议动作。
5. 是否需要审批。
6. 数据缺口。

禁止：

1. 无依据编造项目状态。
2. 把建议写成已经执行的结果。
3. 输出密钥、令牌或敏感配置。
4. 绕过 `approval` 直接执行动作。

## 11. 提示词体系

第一阶段至少维护：

1. 系统角色提示词。
2. 风险分析提示词。
3. 每日站会提示词。
4. 每周周报提示词。
5. 任务建议提示词。

提示词要求：

1. 明确输入字段。
2. 明确输出格式。
3. 强调事实依据。
4. 强调数据不足时说明缺口。
5. 强调高风险动作需要审批。

## 12. 失败降级

| 失败类型 | 降级策略 |
|---|---|
| 模型不可用 | 使用规则或模板文案 |
| 模型超时 | 返回基础摘要并记录超时 |
| 输出格式不合法 | 尝试一次修复，仍失败则降级 |
| 数据不足 | 返回数据缺口，不调用复杂分析 |
| 成本超限 | 使用 fast 模型或模板结果 |

## 13. 决策记录

每次产生建议或自动动作时，必须记录：

1. 决策来源：rule、strategy 或 ai。
2. 输入摘要。
3. 输出内容。
4. 自治级别。
5. 建议动作。
6. 当前状态。
7. 创建和更新时间。

字段定义见 [数据与事件模型](../02-data-and-event-model.md)。

## 14. 成功指标

1. AI 分析输出可读性。
2. 建议采纳率。
3. AI 回复准确性。
4. 平均响应时长。
5. AI 调用成本可控性。
6. 降级成功率。

建议第一阶段目标：

1. 数据不足场景 100% 明确说明缺口。
2. AI 调用失败时 100% 有降级结果。
3. 高风险动作自动执行次数为 0。

## 15. 验收标准

1. 系统能基于项目数据生成风险分析。
2. 系统能生成站会摘要和周报内容。
3. 系统能回答基础项目问答。
4. 系统能对不同动作应用不同自治级别。
5. AI 输出能与项目上下文保持一致。
6. AI 失败时能退回规则或基础文案。

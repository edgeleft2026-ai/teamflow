# 项目运行与协作模块 PRD

## 1. 模块目标

在项目进入运行期后，持续向项目群和项目管理员提供项目状态同步、风险提醒、例行汇报和基础查询能力，帮助团队保持协作节奏并提前发现风险。

## 2. 模块边界

### 2.1 本模块负责

1. 项目群通知。
2. 管理员私聊通知。
3. 逾期提醒。
4. 阻塞提醒。
5. 停滞提醒。
6. 成员负载异常提醒。
7. 每日站会摘要。
8. 每周周报。
9. 风险分析摘要。
10. 项目状态查询。
11. 提醒去重和聚合。
12. 消息路由。

### 2.2 本模块不负责

1. 飞书任务双向同步。
2. 日历联动。
3. 邮件通知。
4. 跨项目管理后台。
5. 复杂订阅规则配置。

## 3. 用户角色

1. 项目管理员：接收重点提醒、初始化回执和需要确认的动作。
2. 项目负责人：关注进度、风险和报告。
3. 项目成员：在群内接收协作通知和基础查询结果。
4. 团队管理者：第一阶段通过群或私聊查看项目状态，不建设独立后台。

## 4. 输入数据

| 数据 | 必填 | 来源 | 说明 |
|---|---|---|---|
| 项目基础信息 | 是 | Project | 项目名称、状态、飞书资源 |
| 成员信息 | 否 | Member | 负责人、成员和角色 |
| 任务状态 | 是 | Task | 提醒和报告的主要依据 |
| Git 活动 | 否 | Git 扫描或导入 | 停滞和技术风险依据 |
| 历史观察 | 否 | Observation | 去重、趋势和报告上下文 |
| 决策记录 | 否 | Decision | 避免重复建议 |

数据不足时，系统必须输出缺少哪些数据，而不是编造结论。

## 5. 提醒能力

### 5.1 逾期提醒

触发条件：

1. 任务已过截止时间。
2. 任务状态不是 `done` 或 `cancelled`。

输出至少包含：

1. 任务名称。
2. 负责人。
3. 截止时间。
4. 逾期天数。
5. 建议动作。

幂等建议：

```text
overdue:{project_id}:{task_id}:{due_date}
```

### 5.2 阻塞提醒

触发条件：

1. 任务状态为 `blocked`。
2. 或任务存在明确阻塞原因。

输出至少包含：

1. 任务名称。
2. 负责人。
3. 阻塞原因。
4. 建议解除方式。

### 5.3 停滞提醒

触发条件：

1. 任务超过配置天数无状态变化。
2. 或关联分支超过配置天数无更新。

输出至少包含：

1. 停滞对象。
2. 最近活动时间。
3. 停滞天数。
4. 建议动作。

### 5.4 成员负载异常提醒

触发条件：

1. 某成员未完成任务数量显著高于团队平均。
2. 或某成员同时承担多个高优先级逾期任务。

输出至少包含：

1. 成员名称或 open_id。
2. 当前任务数量。
3. 团队平均任务数量。
4. 高风险任务列表。
5. 建议调整方向。

## 6. 报告能力

### 6.1 每日站会摘要

内容结构：

1. 昨日完成。
2. 当前进行中。
3. 需要关注。
4. 项目健康度。
5. 今日建议。
6. 数据缺口，如果存在。

### 6.2 每周周报

内容结构：

1. 本周概要。
2. 关键成果。
3. 进行中工作。
4. 风险与问题。
5. 下周计划建议。
6. 团队贡献统计。
7. 数据缺口，如果存在。

### 6.3 风险分析

内容结构：

1. 整体风险等级。
2. 进度风险。
3. 人员风险。
4. 技术风险。
5. 关键风险项。
6. 建议行动。
7. 事实依据。

## 7. 查询能力

### 7.1 结构化指令

第一阶段支持：

1. `/help`
2. `/status`
3. `/tasks`
4. `/risk`
5. `/standup`
6. `/report`

### 7.2 查询规则

1. 群内查询默认返回当前群绑定项目。
2. 私聊查询如果用户只绑定一个项目，默认返回该项目。
3. 私聊查询如果用户关联多个项目，应提示用户指定项目。
4. 查询不到数据时返回空状态说明。

## 8. 消息路由

| 消息类型 | 默认发送位置 | 说明 |
|---|---|---|
| 欢迎和帮助 | 私聊或群 | 取决于触发位置 |
| 普通风险提醒 | 项目群 | 群内公开协作信息 |
| 高价值风险 | 项目群 + 管理员私聊 | 例如严重逾期或多人阻塞 |
| 初始化回执 | 管理员私聊 | 聚焦执行结果 |
| 每日站会 | 项目群 | 默认群通知 |
| 每周周报 | 项目群 + 管理员私聊 | 管理员保留一份 |
| 审批类建议 | 管理员私聊 | 不直接发群执行 |

## 9. 去重和聚合

1. 同一 `dedupe_key` 在配置窗口内只提醒一次。
2. 多个低优先级提醒应聚合成摘要。
3. 高风险提醒可以打破聚合，但仍需避免重复。
4. 去重窗口建议默认 24 小时，可配置。

## 10. 业务规则

1. 所有提醒必须与具体项目绑定。
2. 报告内容必须基于项目数据生成。
3. 数据不足时必须明确说明。
4. 相同事件短时间内重复触发时应去重或聚合。
5. 高风险建议不能直接自动执行，需进入 AI 决策模块判断自治级别。
6. 群消息应避免泄露私聊内容。

## 11. 成功指标

1. 提醒发送成功率。
2. 报告生成成功率。
3. 命令响应成功率。
4. 用户查询响应时延。
5. 重复提醒率。
6. 用户对提醒或建议的采纳率。

建议第一阶段目标：

1. 命令响应在正常依赖下 5 秒内返回基础结果。
2. 同一风险 24 小时内重复提醒次数不超过 1 次。
3. 数据不足时说明覆盖率达到 100%。

## 12. 验收标准

1. 系统能对逾期、阻塞、停滞和负载异常进行提醒。
2. 系统能生成每日站会摘要。
3. 系统能生成每周周报。
4. 用户能在飞书中查询项目状态。
5. 消息不会出现明显重复刷屏。
6. 报告能说明事实依据和数据缺口。
